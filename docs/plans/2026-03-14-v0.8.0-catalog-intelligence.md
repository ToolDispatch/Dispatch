# Dispatch v0.8.0 — Catalog Intelligence Implementation Plan

**Generated:** 2026-03-14
**Branch:** main (post-merge)

---

## 1. Overview

v0.8.0 adds three client-side modules (`llm_client.py`, `stack_scanner.py`) and two server-side features (catalog pipeline, Pro dashboard) that together move Dispatch from a live-API-per-intercept model to a catalog-intelligence model. The LLM adapter decouples classifier/evaluator from the Anthropic SDK so any provider can be used at zero cost on the free tier. Stack detection makes category scoring context-aware by giving the ranker knowledge of the developer's actual project stack. New tool alerts surface high-value additions once per tool without adding latency to the intercept path. The catalog pipeline pre-ranks tools server-side so Pro users get sub-200ms responses. The dashboard closes the feedback loop by showing detection history, bypass rates, and quota usage. All five features must not break the 93-test suite and must never block Claude on failure.

## 4. Integration Points

**llm_client.py is the foundation for all LLM calls.** Both `classify_topic_shift` and `rank_recommendations` call `llm_client.get_client()`. The shared markdown-stripping logic in `complete()` eliminates the repeated pattern duplicated in classifier.py and evaluator.py.

**stack_profile.json flows from stack_scanner through evaluator to ranker.** dispatch.sh triggers `scan_and_save()` on cwd change → writes `stack_profile.json`. preuse_hook.sh reads it and passes stack terms to both `/catalog` (hosted) and `build_recommendation_list(stack_profile=...)` (BYOK).

**state.json is the shared bus between the two hooks.** dispatch.sh writes `last_task_type`, `last_category`, `last_context_snippet`, `last_cwd`. preuse_hook.sh reads all four. `last_cwd` triggers stack rescans.

**seen_alerts.json gates the new tool alert path.** `mark_alert_seen()` writes immediately after deciding to display — so even if display fails, the tool won't alert again. No duplicate alerts across sessions.

**Catalog endpoint replaces live /rank for Pro.** The hosted path hits `/catalog` first. `/rank` stays as fallback if `/catalog` returns empty or non-200. BYOK users are completely unaffected.

**Detection telemetry feeds the dashboard.** Every intercept in hosted mode posts to `/api/detections`. No detection data is collected in BYOK mode. POST is fire-and-forget (2s timeout, `|| true`).

## 5. Risk Flags

**Risk 1 — `openai` package availability.** `install.sh` must check/install `openai` same as `anthropic`. If import fails, `LLMClient` must fall back to Anthropic-only mode. Wrap `import openai` in try/except inside `LLMClient.__init__`.

**Risk 2 — Free-tier LLM quality regression.** `llama-3.1-8b-instruct:free` produces valid JSON less reliably than Haiku. `complete()` should attempt `json.loads()` in try/except and return raw text on failure rather than `""` — let callers handle malformed JSON via existing fallbacks.

**Risk 3 — Stack scanner runs at hook time.** cwd-change rescan in dispatch.sh runs inline during UserPromptSubmit. Wrap `scan_and_save()` call in `|| true`. Add 1-second timeout using Python `signal` module inside `detect_stack()`.

**Risk 4 — /catalog empty for new installs.** Until first daily cron runs, `tool_catalog` is empty. Three-tier fallback required: `/catalog` → live `/rank` → BYOK `build_recommendation_list()`.

**Risk 5 — Detection volume at scale.** 1,000 users × 10 calls/session = 10,000 rows/day. `idx_detections_user_ts` covers per-user queries. Add `created_at < NOW() - INTERVAL '90 days'` cleanup in cron job.

**Risk 6 — preuse_hook.sh budget with new additions.** v0.8.0 budget: bypass check (~5ms) + alert check (≤1.5s) + catalog call (~200ms) + detection POST (~2s fire-and-forget) = ~4.2s worst case. Net improvement over v0.7.0's ~5s because /catalog replaces ~4s /rank call. Alert check timeout reduced to 1.5s for safety.

**Risk 7 — seen_alerts.json race condition.** Two rapid tool calls could both read seen_alerts before either writes → same alert surfaces twice. Acceptable for v0.8.0 given low frequency of new tool additions.
